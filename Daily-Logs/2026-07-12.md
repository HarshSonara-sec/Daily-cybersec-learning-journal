## Hack The Box Academy – Introduction to Networking

### MAC Address (Media Access Control)

* A MAC (Media Access Control) address is a unique **48-bit (6-byte)** hardware identifier assigned to a Network Interface Card (NIC).
* Operates at **Layer 2 (Data Link Layer)** of the OSI model.
* Used for communication between devices on the same Local Area Network (LAN).
* Written in hexadecimal format (e.g., `00:1A:2B:3C:4D:5E`).
* The first 24 bits represent the **Organizationally Unique Identifier (OUI)**, identifying the hardware manufacturer.
* The last 24 bits uniquely identify the network interface.
* Switches use MAC addresses to forward Ethernet frames to the correct destination.
* Unlike IP addresses, MAC addresses are hardware-based identifiers and typically remain constant unless intentionally changed (MAC spoofing).

#### Security Notes

* **MAC Spoofing:** An attacker changes their device's MAC address to impersonate another device or bypass MAC-based filtering.
* **MAC Flooding:** An attacker overwhelms a switch's MAC address table with fake entries, potentially causing the switch to broadcast traffic to all ports and enabling packet sniffing.
* MAC filtering alone is **not** considered a strong security mechanism because MAC addresses can be spoofed.

## ARP (Address Resolution Protocol)

* ARP maps an **IPv4 address** to its corresponding **MAC address** on a local network.
* Operates between **Layer 2 and Layer 3**, allowing IP communication over Ethernet.
* Before sending data on a LAN, a device checks its **ARP cache** for the destination MAC address.
* If no entry exists:

  1. The sender broadcasts an **ARP Request** asking, "Who has this IP?"
  2. The device with the matching IP responds with an **ARP Reply** containing its MAC address.
  3. The sender stores the mapping in the ARP cache for future communication.

### ARP Cache

* Stores recently resolved IP-to-MAC mappings.
* Reduces unnecessary ARP broadcasts.
* Entries expire after a period of time and are refreshed as needed.

#### Security Notes

* **ARP Spoofing (ARP Poisoning):** An attacker sends forged ARP replies to associate their MAC address with another device's IP address (often the default gateway).
* This can position the attacker between two communicating devices, creating a **Man-in-the-Middle (MITM)** attack.
* Consequences include:

  * Intercepting network traffic
  * Modifying transmitted data
  * Stealing credentials
  * Session hijacking
  * Redirecting users to malicious destinations
  * Causing denial-of-service by disrupting normal traffic flow

#### Common Defensive Measures

* Static ARP entries (where appropriate)
* Dynamic ARP Inspection (DAI)
* Port Security
* VLAN segmentation
* HTTPS, SSH, and VPNs to encrypt intercepted traffic
* Monitoring for duplicate or suspicious ARP entries

---

## Key Takeaways

* **IP addresses** identify devices logically (**Layer 3**).
* **MAC addresses** identify network interfaces physically (**Layer 2**).
* **ARP** connects Layer 3 and Layer 2 by resolving IP addresses into MAC addresses.
* Communication on a LAN depends on successful ARP resolution.
* Understanding MAC addresses and ARP is fundamental for network troubleshooting, packet analysis, penetration testing, and detecting attacks such as MAC spoofing and ARP spoofing.

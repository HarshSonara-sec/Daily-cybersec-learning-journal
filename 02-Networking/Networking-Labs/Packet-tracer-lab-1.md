# CCNA Lab 1 – Cisco Packet Tracer

## Objective

Set up the Cisco Packet Tracer environment and recreate a basic enterprise network topology.

## Practical Tasks Completed

* Installed and configured Cisco Packet Tracer.
* Opened the provided CCNA lab file.
* Explored the Packet Tracer interface and preferences.
* Customized CLI appearance for better readability.
* Added the required network devices:

  * Cisco 2911 Routers
  * Cisco Catalyst 2960 Switches
  * Cisco ASA 5505 Firewalls
  * PCs
  * Servers
  * Attacker Laptop
* Renamed all devices according to the network diagram (R1, R2, SW1, SW2, FW1, FW2, PC1, PC2, SRV1, SRV2).
* Connected all devices using the automatic connection tool.
* Used the **Ctrl + Click** shortcut to speed up multiple cable connections.
* Successfully recreated the network topology from the course diagram.

## Skills Practiced

* Navigating Cisco Packet Tracer
* Adding and organizing network devices
* Creating network topologies
* Device naming conventions
* Connecting devices with appropriate cables

## Key Takeaways

* Packet Tracer provides a virtual environment for practicing Cisco networking without physical hardware.
* Proper device naming and organized topology make network management easier.
* Building network diagrams is the first step before configuring routers, switches, and firewalls.

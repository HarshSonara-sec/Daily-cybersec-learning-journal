# CCNA Lab 2 – Connecting Network Devices

**Date:** July 10, 2026

## Objective

Practice selecting and connecting network devices using the correct cable types based on device type and connection distance.

## Practical Tasks Completed

* Built the lab topology in Cisco Packet Tracer.
* Connected PCs and servers to switches using **copper straight-through** cables.
* Connected switches to other switches using **copper crossover** cables.
* Connected switches to routers using **copper straight-through** cables.
* Connected routers to each other using the appropriate cable type based on distance:

  * Copper crossover for short-distance connections.
  * Multimode fiber for medium-distance connections.
  * Single-mode fiber for long-distance connections.
* Verified all network devices were connected according to the lab topology.

## Skills Practiced

* Identifying the correct cable type for different network devices.
* Understanding when to use straight-through, crossover, and fiber optic cables.
* Applying cable distance limitations in network design.
* Building and organizing network topologies in Cisco Packet Tracer.

## Key Takeaways

* **Straight-through cables** connect different device types (PC ↔ Switch, Switch ↔ Router).
* **Crossover cables** connect similar device types (Switch ↔ Switch, Router ↔ Router).
* **Multimode fiber** is suitable for medium-distance connections, while **single-mode fiber** is used for long-distance links.
* Selecting the correct cable type is essential for reliable network communication.
